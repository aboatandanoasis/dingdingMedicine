用药提醒小程序

一款专为老年人设计的极简用药管理工具。

1. 项目背景
随着老龄化社会到来，越来越多的老人需要长期服药管理慢性疾病。然而在实际生活中，他们面临着诸多困扰：
- 记性差：经常忘记自己今天吃没吃药，有时重复服用，有时又漏服
- 药品多：降压药、降糖药、钙片……种类繁多，容易搞混服用时间和用量
- 操作难：智能手机操作复杂，很多 APP 功能繁杂，老人不会用
- 无人知：子女不在身边，无法了解父母的用药情况
本项目旨在通过极简的设计理念，帮助老年人轻松管理日常用药。

2. 需求分析
（1）目标用户
- 主要用户：60岁以上需要长期服药的老年人
- 次要用户：照顾老人用药的子女、护工
（2）核心诉求
- 简单易用：大字体、大按钮、一键操作
- 状态清晰：红/黄/绿灯直观显示服药状态
- 避免重复/遗漏：记录每次确认，历史可追溯
- 低学习成本：无需注册登录，打开即用

3. 功能规划
当前版本 (v1.0 Demo)
✅ 药品管理：添加/编辑/删除药品信息
✅ 状态切换：已吃/未到时间/过时未吃
✅ 一键确认：点击大按钮即完成当日时段服药确认
✅ 历史记录：自动记录服药时间，可随时查看
后续迭代
- 分时段管理：早/中/晚分时段记录，同时段多次修改保留最终结果
- 订阅消息提醒：手机微信端定时推送服药通知
- 多用户系统：子女端可远程操作提醒和查看老人用药情况

4. 技术方案
（1）技术栈
- 前端	微信小程序原生开发
- 后端	微信云开发 (Serverless)
- 数据库	云开发数据库
- 存储	云存储

（3）数据结构
- 药品表 (本地存储)
javascript
{
  id: Number,          // 唯一标识
  name: String,        // 药品名称
  description: String, // 外观描述
  dosage: String,      // 每次用量
  unit: String,        // 单位（片/粒/颗等）
  taken: Boolean,      // 今日是否已服用
  status: String,      // 状态：done/pending/overdue
  image: String        // 药品图片
}
- 服药记录表 (云数据库)
javascript
{
  medicineId: Number,    // 药品ID
  medicineName: String,  // 药品名称
  dosage: String,        // 用量
  unit: String,          // 单位
  takeTime: Number,      // 服药时间戳
  takeTimeStr: String    // 格式化时间 (HH:MM)
}

5. 项目亮点
（1）极简设计理念
字体最大 56rpx（约 28px），远超常规 APP
核心按钮高度 220rpx（约 110px），方便点击
全程只需"点一下"，无复杂交互
（2）无障碍体验
高对比度配色，弱视老人也能看清
状态用颜色 + 图标 + 文字三重表达
问候语增加情感温度
（3）AI 辅助开发

需求分析与功能规划
界面设计与交互方案
代码开发与问题调试
文档撰写与产品说明

# 云开发 quickstart

这是云开发的快速启动指引，其中演示了如何上手使用云开发的三大基础能力：

- 数据库：一个既可在小程序前端操作，也能在云函数中读写的 JSON 文档型数据库
- 文件存储：在小程序前端直接上传/下载云端文件，在云开发控制台可视化管理
- 云函数：在云端运行的代码，微信私有协议天然鉴权，开发者只需编写业务逻辑代码
